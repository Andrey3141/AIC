## 📖 О проекте

**AIC v2.0** — это система автоматизированной разработки, где 8 AI-сотрудников взаимодействуют через локальную LLM (LM Studio). В отличие от первой версии (которая использовала DeepSeek через браузер), v2.0 работает полностью локально, быстрее и надежнее.

### 🎯 Основная идея

Вы даёте задачу → **Распределитель** анализирует и направляет → сотрудники выполняют свои роли → результат возвращается вам.

### Роли в команде

| Роль | Иконка | Ответственность |
|------|--------|-----------------|
| **Распределитель** | 🛰️ | Анализирует сообщения, решает кому отправить (ignore/save/send) |
| **Начальник** | 👨‍💼 | Решает конфликты, принимает финальные решения |
| **Аналитик** | 📑 | Формирует ТЗ, уточняет требования, ведёт документацию |
| **Главный разработчик** | 🔝👨‍💻 | Проектирует архитектуру, проверяет код, управляет разработкой |
| **Главный тестировщик** | 🔝🐞 | Определяет стратегию тестирования, подтверждает баги |
| **Рядовой разработчик** | 👨‍💻 | Пишет код, исправляет баги |
| **Рядовой тестировщик** | 🐞 | Пишет тесты, находит баги |
| **Пользователь** | 👨‍💼 | Даёт задачу, получает результат |

### Почему v2.0 лучше v1?

| Аспект | AIC v1 (DeepSeek) | AIC v2 (LM Studio) |
|--------|-------------------|---------------------|
| **Зависимости** | Браузер + Selenium | Только LM Studio |
| **Скорость** | Медленно (ожидание таймеров) | Быстро (прямые запросы) |
| **Надёжность** | Зависит от UI DeepSeek | Стабильный API |
| **Настройка** | 5 вкладок, ручное переключение | Автоматическое распределение |
| **Роли** | 5 фиксированных | 8 гибких (через конфиг) |
| **Промпты** | .txt | .yaml (структурированные) |

### Действия распределителя

| Действие | Описание |
|----------|----------|
| `ignore` | Роль не имеет отношения к сообщению |
| `save` | Сохранить в контекст (полезная информация) |
| `send` | Требуется ответ / действие от роли |

## 📥 Установка и запуск

### ⚡ Быстрый старт

```bash
# 1. Клонируйте или скопируйте файлы проекта
cd aic_v2

# 2. Установите зависимости (только стандартная библиотека)
# Дополнительные модули не требуются! Используется:
# - tkinter (встроен в Python)
# - requests (установите при необходимости)
# - pyyaml (установите при необходимости)

pip install requests pyyaml

# 3. Запустите LM Studio
# - Скачайте LM Studio с официального сайта
# - Загрузите модель (например, Qwen2.5-Coder-7B)
# - Откройте вкладку "Developer" → "Start Server"

# 4. Запустите AIC
python main.py
```

### 📦 Требования

- Python 3.8 или выше
- LM Studio (с запущенным локальным сервером на порту 1234)
- Модель: Qwen2.5-Coder-7B / qwen3-coder-30b (или любая совместимая)

### 🔧 Настройка перед запуском

1. **Установите LM Studio** → загрузите модель → запустите сервер
2. **Проверьте конфиг** (`config.json`):
   ```json
   {
       "lm_studio_url": "http://localhost:1234/v1/chat/completions",
       "model": "qwen/qwen3-coder-30b",
       "temperature": 0.3,
       "max_tokens": 32768
   }
   ```
3. **Убедитесь, что папка `prompts/` содержит все `.yaml` файлы ролей**

### 🚀 Запуск

```bash
python main.py
```

После запуска:
1. Нажмите **ПОДКЛЮЧИТЬСЯ** → проверка соединения с LM Studio
2. Нажмите **⚙️ ФОРМАТ ЗАПРОСА** → выберите язык, платформу, IDE
3. Введите задачу в формате:
   ```
   Язык: Python
   Платформа: Desktop
   IDE: VS Code
   Задача: Реализуй калькулятор с операциями +, -, *, /
   ```
4. Нажмите **📤 ОТПРАВИТЬ** → система запустит пайплайн

## 🔧 Решение проблем

> [ERR-A001] - Не подключается к LM Studio

### Присутствует в версиях: [v2.0.0]
### Симптомы: При нажатии "ПОДКЛЮЧИТЬСЯ" статус остаётся OFFLINE
### Причина: LM Studio не запущен или сервер не активен
### Решение: 
1. Откройте LM Studio
2. Загрузите модель (например, Qwen2.5-Coder-7B)
3. Перейдите на вкладку "Developer"
4. Нажмите "Start Server" (порт 1234)
5. В AIC нажмите "ПОДКЛЮЧИТЬСЯ" снова

---

> [ERR-A002] - Ошибка "Модель не найдена"

### Присутствует в версиях: [v2.0.0]
### Симптомы: В логе ошибка "Model not found"
### Причина: В config.json указано имя модели, которая не загружена в LM Studio
### Решение:
1. Откройте LM Studio → вкладка "Models"
2. Посмотрите точное имя загруженной модели
3. Обновите `model` в `config.json`
4. Перезапустите AIC

---

> [ERR-A003] - Распределитель не отправляет сообщение

### Присутствует в версиях: [v2.0.0]
### Симптомы: Сообщение повисает, нет ответа от сотрудников
### Причина: В ответе DISTRIBUTOR нет строки с `send` для какой-либо роли
### Решение:
1. Проверьте формат сообщения (должен быть "Язык: ... Платформа: ...")
2. Убедитесь, что промпт `Distributor.yaml` корректен
3. Посмотрите лог: там будет вывод DISTRIBUTOR
4. При необходимости отредактируйте `Distributor.yaml`

---

> [ERR-A004] - Ошибка чтения YAML файла

### Присутствует в версиях: [v2.0.0]
### Симптомы: В логе "Файл не найден" или ошибка парсинга
### Причина: Отсутствует файл промпта в папке `prompts/`
### Решение:
1. Проверьте, что все файлы присутствуют:
   - Distributor.yaml
   - Boss.yaml
   - Analyst.yaml
   - Chief_developer.yaml
   - Chief_tester.yaml
   - Ordinary_developer.yaml
   - Ordinary_tester.yaml
2. Убедитесь, что в файле есть поле `prompt: |`
3. Скопируйте эталонные файлы из репозитория

---

> [ERR-A005] - Таймаут ожидания ответа

### Присутствует в версиях: [v2.0.0]
### Симптомы: Лог "Таймаут ожидания ответа" после долгого ожидания
### Причина: Модель генерирует ответ слишком долго (>30 минут)
### Решение:
1. Увеличьте `max_tokens` в `config.json`
2. Используйте менее тяжёлую модель
3. Увеличьте таймаут в `lm_studio_client.py` (параметр `timeout`)
