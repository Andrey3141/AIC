# FAQ.md — AIC v2.0

## 🔌 ПОДКЛЮЧЕНИЕ И ЗАПУСК

> [ERR-A001] - Не подключается к LM Studio

### Присутствует в версиях: [v2.0.0]
### Симптомы: При нажатии "ПОДКЛЮЧИТЬСЯ" статус остаётся OFFLINE, в логе ошибка "Connection refused"
### Причина: LM Studio не запущен или сервер не активен
### Решение:
1. Откройте LM Studio
2. Загрузите модель (например, Qwen2.5-Coder-7B)
3. Перейдите на вкладку "Developer"
4. Нажмите "Start Server" (порт 1234)
5. Проверьте, что в конфиге указан верный URL: `http://localhost:1234/v1/chat/completions`
6. В AIC нажмите "ПОДКЛЮЧИТЬСЯ" снова

---

> [ERR-A002] - Ошибка "Model not found"

### Присутствует в версиях: [v2.0.0]
### Симптомы: В логе ошибка "Model not found" или "404 Not Found"
### Причина: В config.json указано имя модели, которая не загружена в LM Studio
### Решение:
1. Откройте LM Studio → вкладка "Models"
2. Посмотрите точное имя загруженной модели (например, `qwen/qwen3-coder-30b`)
3. Откройте `config.json` и обновите поле `"model"`
4. Перезапустите AIC
5. Нажмите "ПОДКЛЮЧИТЬСЯ"

---

> [ERR-A003] - Приложение не запускается (ошибка импорта)

### Присутствует в версиях: [v2.0.0]
### Симптомы: Терминал выводит "ModuleNotFoundError: No module named 'requests'" или "No module named 'yaml'"
### Причина: Отсутствуют необходимые зависимости
### Решение:
```bash
pip install requests pyyaml
```

## 💬 РАБОТА С СОТРУДНИКАМИ

> [ERR-B001] - Распределитель не отправляет сообщение

### Присутствует в версиях: [v2.0.0]
### Симптомы: Сообщение повисает, нет ответа от сотрудников, лог показывает DISTRIBUTOR → parsed: {}
### Причина: В ответе DISTRIBUTOR нет строки с `send` для какой-либо роли
### Решение:
1. Проверьте формат сообщения (должен быть "Язык: ... Платформа: ... IDE: ... Задача: ...")
2. Убедитесь, что промпт `prompts/Distributor.yaml` корректен
3. Посмотрите лог: там будет вывод DISTRIBUTOR (строка "PIPE INPUT")
4. При необходимости отредактируйте `Distributor.yaml`, добавив примеры

---

> [ERR-B002] - Сотрудник отвечает не по роли

### Присутствует в версиях: [v2.0.0]
### Симптомы: Например, АНАЛИТИК начинает писать код или ТЕСТИРОВЩИК принимает архитектурные решения
### Причина: Промпт роли не содержит чётких запретов или модель "фантазирует"
### Решение:
1. Откройте соответствующий `.yaml` файл в папке `prompts/`
2. Усилите раздел "ЧТО ТЫ НЕ ДЕЛАЕШЬ"
3. Уменьшите `temperature` в `config.json` (например, с 0.6 до 0.3)
4. Перезапустите AIC

---

> [ERR-B003] - Цикл: один и тот же сотрудник отвечает несколько раз подряд

### Присутствует в версиях: [v2.0.0]
### Симптомы: В чате видно 3+ сообщений от одной роли без участия других
### Причина: Распределитель не видит явного адресата и назначает `send` той же роли
### Решение:
1. Убедитесь, что в ответе сотрудника есть строка `=== КОМУ === [РОЛЬ]`
2. Проверьте, что роль не отправляет сообщение сама себе
3. Отредактируйте `Distributor.yaml`, добавив правило: "Отправитель НЕ может иметь send"

## 📁 ФАЙЛЫ И КОНФИГУРАЦИЯ

> [ERR-C001] - Ошибка чтения YAML файла

### Присутствует в версиях: [v2.0.0]
### Симптомы: В логе "Файл не найден: Boss.yaml" или ошибка парсинга YAML
### Причина: Отсутствует файл промпта в папке `prompts/`
### Решение:
1. Проверьте, что все файлы присутствуют:
   - `Distributor.yaml`
   - `Boss.yaml`
   - `Analyst.yaml`
   - `Chief_developer.yaml`
   - `Chief_tester.yaml`
   - `Ordinary_developer.yaml`
   - `Ordinary_tester.yaml`
2. Убедитесь, что в файле есть структура:
   ```yaml
   prompt: |
     текст промпта
   ```
3. Скопируйте эталонные файлы из архива проекта

---

> [ERR-C002] - Не сохраняются логи

### Присутствует в версиях: [v2.0.0]
### Симптомы: Папка `logs/` пустая или отсутствует
### Причина: Нет прав на запись в текущей директории
### Решение:
1. Создайте папку `logs` вручную
2. Проверьте права доступа: `chmod 755 logs` (Linux/Mac)
3. При закрытии приложения логи сохраняются автоматически

---

> [ERR-C003] - config.json повреждён

### Присутствует в версиях: [v2.0.0]
### Симптомы: При запуске ошибка JSON decode error или приложение использует настройки по умолчанию
### Причина: В файл попали невалидные символы или запятые
### Решение:
1. Закройте AIC
2. Откройте `config.json` в любом редакторе
3. Проверьте валидность на https://jsonlint.com
4. Удалите файл — приложение создаст новый с настройками по умолчанию

## ⚡ ПРОИЗВОДИТЕЛЬНОСТЬ

> [ERR-D001] - Таймаут ожидания ответа

### Присутствует в версиях: [v2.0.0]
### Симптомы: Лог "Таймаут ожидания ответа" после долгого ожидания (>30 минут)
### Причина: Модель генерирует ответ слишком долго или зависла
### Решение:
1. Увеличьте `max_tokens` в `config.json` (до 32768)
2. Используйте менее тяжёлую модель (например, Qwen2.5-Coder-7B вместо 30B)
3. Увеличьте таймаут в `lm_studio_client.py` (параметр `timeout=1800`)

---

> [ERR-D002] - Модель потребляет всю память

### Присутствует в версиях: [v2.0.0]
### Симптомы: Компьютер тормозит, LM Studio вылетает
### Причина: Модель слишком большая для вашего ОЗУ
### Решение:
1. Используйте квантованную версию модели (Q4_K_M вместо FP16)
2. Уменьшите `max_tokens` до 4096
3. Попробуйте модель меньшего размера (7B вместо 30B)

## 🔄 МИГРАЦИЯ С v1 НА v2

> [ERR-F001] - Можно ли использовать старые промпты (.txt) в v2?

### Присутствует в версиях: [v2.0.0]
### Симптомы: Приложение не видит файлы .txt в папке `prompts/`
### Причина: v2 использует YAML формат, а не TXT
### Решение:
1. Конвертируйте каждый `.txt` в `.yaml`:
   ```yaml
   prompt: |
     (содержимое вашего .txt файла)
   ```
2. Сохраните с тем же именем (например, `Boss.yaml`)
3. Удалите старые `.txt` файлы

---

> [ERR-F002] - Куда делись окна браузера? (v1 vs v2)

### Присутствует в версиях: [v2.0.0]
### Симптомы: Нет кнопок "REFRESH", "ПЕРЕКЛЮЧИТЬ", управления вкладками
### Причина: v2 не использует Selenium и браузер — всё работает через LM Studio API
### Решение: Ничего делать не нужно. В v2 не требуется открывать вкладки DeepSeek.
