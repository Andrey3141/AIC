## 🗺️ План развития

### 🔜 v2.1.0 — Улучшение оркестрации
- [ ] Добавить ограничение на максимальное количество шагов в пайплайне
- [ ] Реализовать сохранение состояния (checkpoint) для возможности восстановления
- [ ] Добавить возможность ручного вмешательства (прервать пайплайн, перенаправить сообщение)
- [ ] Улучшить парсинг ответа DISTRIBUTOR (поддержка альтернативных форматов)

### 🔜 v2.2.0 — Расширение ролей
- [ ] Добавить роль **DevOps** (деплой, CI/CD)
- [ ] Добавить роль **Security** (аудит безопасности кода)
- [ ] Добавить роль **Tech Writer** (отдельная роль для документации)
- [ ] Сделать систему ролей полностью конфигурируемой (добавление/удаление через config.json)

### 🔜 v2.3.0 — Интеграции
- [ ] Поддержка **Ollama** (альтернатива LM Studio)
- [ ] Поддержка **OpenAI API** (GPT-4, GPT-3.5)
- [ ] Поддержка **Anthropic Claude API**
- [ ] Экспорт результатов в **GitHub** (создание PR с кодом)

### 🔜 v2.4.0 — Тестирование и качество
- [ ] Автоматический запуск pytest (как в v1)
- [ ] Интеграция с **SonarQube** (статический анализ кода)
- [ ] Генерация отчёта о покрытии тестами
- [ ] Режим "Self-test" (проверка корректности работы всех ролей)

### 🔜 v2.5.0 — UX улучшения
- [ ] Тёмная тема интерфейса
- [ ] Настраиваемые цвета для каждой роли
- [ ] Поиск по истории сообщений
- [ ] Экспорт диалога в HTML/Markdown
- [ ] Drag & Drop для прикрепления файлов

### 🔜 v3.0.0 — Клиент-серверная архитектура
- [ ] Разделение на backend (FastAPI) и frontend (React/Tkinter)
- [ ] Поддержка нескольких пользователей одновременно
- [ ] Веб-интерфейс вместо Tkinter
- [ ] Docker контейнер для простого развёртывания
