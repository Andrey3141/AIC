##  О проекте

**CyberDial** — это игровой симулятор рабочего стола телефона с полноценным встроенным Telegram-клиентом. Приложение создает полную иллюзию использования реального смартфона: **единый вертикальный экран** с виджетами, иконки приложений, папки, нижняя панель и даже мессенджер с живыми диалогами.

### Что нового в v1.4.0?

| Было (v1.3.0) | Стало (v1.4.0) |
|---------------|----------------|
| 2 страницы с горизонтальным листанием | **Единый вертикальный экран** |
| ViewPager2 + индикатор страниц | **Мультитайп RecyclerView** |
| Чёрные квадраты для недоступных приложений | **Видна иконка + затемнение + замок** |
| XML-анимации scale_in/scale_out | **Ripple + программный scale** |
| Смешанная архитектура | **MVVM (ViewModel + Repository)** |

### Почему это круто?

| Обычные лаунчеры | CyberDial |
|------------------|-----------|
| Требуют установку как системное приложение | Работает как обычное приложение-игра |
| Скучный дизайн | Стилизация под телефон в игровой форме |
| Нет виджетов | Красочные виджеты с анимацией |
| Обычные папки | Папки с миниатюрами 3x3 |
| Нет мессенджера | Полноценный Telegram с диалогами |

##  Telegram бот

### Как работает диалог с Мамулей 

1. **Запуск Telegram** → открывается список чатов
2. **Таймер 60 секунд** → виджет показывает обратный отсчёт
3. **Случайный выбор роли** (50% мама / 50% мошенник)
4. **Кнопки выбора** → пользователь выбирает готовый ответ
5. **Бот отвечает** → диалог продолжается по сценарию
6. **Завершение** → после последнего сообщения диалог заканчивается

##  Решение проблем

<details>
<summary> Не компилируется проект</summary>

1. Проверьте версию Gradle: `./gradlew --version`
2. Синхронизируйте проект: `File → Sync Project with Gradle Files`
3. Очистите кэш: `Build → Clean Project`, затем `Build → Rebuild Project`
</details>

<details>
<summary> Не отображаются иконки приложений</summary>

- Убедитесь, что все ресурсы (`ic_*.xml`) присутствуют в папке `drawable/`
- Проверьте `AppsRepository.kt` — правильные ли имена ресурсов?
</details>

<details>
<summary> Не открывается папка</summary>

- Убедитесь, что `isFolder = true` у модели
- Проверьте, что `appsInFolder` не равен `null`
</details>

<details>
<summary> Заблокированное приложение не имеет затемнения</summary>

- Проверьте `isAvailable = false` у модели
- В `AppViewHolder` должна применяться `alpha = 0.5f`
</details>

<details>
<summary> Таймер Мамули не обновляется</summary>

- Проверьте `WidgetsViewHolder.startMamulyaTimer()`
- Убедитесь, что `onDestroy()` отключает Handler
</details>

# CHANGELOG.md

## [v1.4.0] - 2026-04-24

### Breaking Changes
- **Удалён ViewPager2** — больше нет горизонтального листания страниц
- **Удалён HomeScreenPagerAdapter** — класс полностью удалён
- **Удалены индикаторы страниц** (`page_indicator_active.xml`, `page_indicator_inactive.xml`)
- **Изменена навигация** — теперь единый вертикальный скролл вместо двух страниц

### Added
- **Мультитайп RecyclerView** — единая вертикальная лента с разными типами элементов
- **MainScreenAdapter** — управляет отображением виджетов и приложений в одном списке
- **WidgetsViewHolder** — отдельный холдер для 4 виджетов (погода, музыка, календарь, Мамуля)
- **HomeScreenViewModel** — MVVM архитектура, управляет состоянием экрана
- **AppsRepository** — централизованный источник данных о приложениях
- **WidgetsData** — data class для хранения состояния виджетов
- **FolderDialogFragment** — BottomSheetDialog для отображения папок
- **FolderAdapter** — адаптер для сетки 3×3 внутри папок
- **Визуальная индикация недоступных приложений** — иконка + полупрозрачное затемнение (alpha 0.5) + иконка замка
- **Ripple-эффект** для всех кликабельных элементов
- **Программная анимация масштаба** (0.95 → 1.0) вместо XML-анимаций
- **Новая статусная панель** в `activity_main.xml` (время, сигнал, батарея)

### Removed
- `HomeScreenPagerAdapter.kt` — больше не нужен
- `res/drawable/page_indicator_active.xml` — индикатор страниц удалён
- `res/drawable/page_indicator_inactive.xml` — индикатор страниц удалён
- `res/anim/scale_in.xml` — заменён на программную анимацию
- `res/anim/scale_out.xml` — заменён на программную анимацию
- Второй фрагмент (все приложения теперь на одном экране)

### Changed
- **HomeScreenFragment полностью переписан**:
  - Убран ViewPager2
  - Добавлен RecyclerView с GridLayoutManager(5)
  - Реализован spanSizeLookup (виджеты на 5 колонок)
  - Добавлено наблюдение за LiveData из ViewModel
- **AppModel расширен**:
  - Добавлены поля `isAvailable` (доступность)
  - Добавлено поле `isFolder` (флаг папки)
  - Добавлено поле `appsInFolder` (список для папок)
  - Добавлено поле `packageName` (для реального запуска)
- **AppIconAdapter заменён на MainScreenAdapter** (старый класс остался, но не используется)
- **Анимации нажатий**:
  - Вместо XML → программный scale с duration 100ms
  - Добавлен `android:foreground="?attr/selectableItemBackgroundBorderless"` для ripple
- **Статусная панель** перенесена из фрагмента в `activity_main.xml`
- **Нижняя панель** теперь фиксирована внизу фрагмента (не скроллится)

### Fixed
- **Чёрные квадраты недоступных приложений** — теперь видна иконка приложения (полупрозрачная) с замком
- **Дублирование приложения Telegram** — в папке Соцсети и в отдельной сетке (требуется ручное исправление)
- **Обрезание названий приложений** — увеличена ширина иконки до 60dp (частично решено)
- **Утечка памяти в таймере Мамули** — добавлен `stopMamulyaTimer()` в `onDestroy`

### Performance
- Оптимизирована отрисовка — единый RecyclerView вместо ViewPager2 + два RecyclerView
- Уменьшено количество создаваемых вьюх (виджеты не пересоздаются при скролле)
- Использование spanSizeLookup вместо вложенных RecyclerView

### Architecture
- **Внедрена MVVM**:
  - `HomeScreenViewModel` — бизнес-логика
  - `AppsRepository` — источник данных
  - `HomeScreenFragment` — только отображение
  - `LiveData` — реактивное обновление UI
- **Разделение ответственности** каждая сущность теперь делает только своё дело

> [Error-8] - Дублируется приложение Telegram в сетке приложений

### Присутствует в версиях: [v1.4.0] - текущая
### Симптомы: Telegram отображается дважды — внутри папки "Соцсети" и отдельной иконкой в сетке
### Причина: Ошибка в `AppsRepository.kt` — приложение добавлено и в папку, и в отдельный список
### Решение: Удалите дубликат из одного из списков (рекомендуется оставить только внутри папки или только отдельно, по выбору)

---

> [Error-7] - Чёрный экран на секунду при открытии Telegram

### Присутствует в версиях: [v1.0.0] - текущая
### Симптомы: При нажатии на Telegram экран становится чёрным на долю секунды, затем появляется SplashScreen
### Причина: Стандартное поведение при смене активности
### Решение: Это не ошибка, так задумано для имитации загрузки мессенджера

---

> [Error-6] - Приложение не устанавливается на Android версии ниже 7.0

### Присутствует в версиях: [v1.3.0] - текущая
### Симптомы: Ошибка "Приложение не установлено" или "Не удалось проанализировать пакет"
### Причина: Начиная с версии v1.3.0 требуется Android API 24 (Android 7.0) и выше
### Решение: Обновите прошивку устройства до Android 7.0 или выше

---

> [Error-5] - Нет поля ввода текста в чате с Мамулей

### Присутствует в версиях: [v1.2.0] - текущая
### Симптомы: В диалоге с Мамулей нельзя написать свой текст, только нажимать на кнопки
### Причина: Это особенность сценарного диалога с фиксированными вариантами ответов
### Решение: Выбирайте один из предложенных вариантов ответа под сообщениями бота

---

> [Error-4] - Мамуля пишет не сразу

### Присутствует в версиях: [v1.1.0] - текущая
### Симптомы: После нажатия на Telegram нужно ждать 60 секунд, прежде чем придёт первое сообщение
### Причина: Сюжетный таймер для создания напряжения
### Решение: Дождитесь окончания таймера на виджете "Мамуля скоро напишет..."

---

> [Error-3] - При нажатии на заблокированное приложение ничего не происходит

### Присутствует в версиях: [v1.0.0] - текущая
### Симптомы: Нажатие на серую иконку с замком не открывает приложение
### Причина: Это игровая механика, а не ошибка. Только активные (цветные) приложения работают
### Решение: В версии v1.4.0 заблокированные приложения показывают иконку с затемнением и замком — нажатие показывает Toast "Приложение заблокировано". Разблокировка появится в следующих версиях

---

> [Error-2] - Обрезаются названия приложений

### Присутствует в версиях: [v1.0.0] - текущая
### Симптомы: "Развлечения" отображается как "Развле...", "WhatsApp" как "Whats..."
### Причина: Ширина иконки (56dp) не вмещает длинные названия в одну строку
### Решение: В версии v1.4.0 ширина увеличена до 60dp (частичное решение). Полное решение — в плане (разрешить 2 строки)

---

> [Error-1] - Бейдж на иконке Telegram не отображается

### Присутствует в версиях: [v1.0.0] - [v1.2.0]
### Симптомы: На иконке Telegram нет красного кружка с числом непрочитанных сообщений
### Причина: Ошибка в механизме обновления бейджа между активностями
### Решение: Обновитесь до версии v1.3.0 или выше, проблема исправлена
