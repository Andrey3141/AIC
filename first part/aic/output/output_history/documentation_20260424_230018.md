О проекте
Калькулятор с интерфейсом в стиле Excel (сетка кнопок + строка формул), возможностью голосового ввода на русском языке и настройкой цветовой схемы кнопок. Работает полностью оффлайн благодаря интеграции с Vosk.
Ключевые особенности
Особенность Описание
Excel-стиль Строка формул для ввода выражений + сетка кнопок
Голосовой ввод Распознавание речи на русском языке оффлайн
Настройка цветов Изменение цвета цифровых кнопок и кнопок операций
MVVM архитектура Строгое разделение логики, интерфейса и данных
Безопасные вычисления AST-валидация без использования eval()
Поддержка скобок Полноценные математические выражения

Использование
Клавиатура и мышь
Действие Как выполнить
Ввести число Нажать кнопку с цифрой

Голосовое управление
Нажмите кнопку Голос — начнётся запись (кнопка станет красной)
Скажите команду, например: "пять плюс три"
Нажмите Голос снова или подождите 2 секунды тишины — распознавание завершится
Распознанное выражение появится в строке формул
Примеры голосовых команд
Вы сказали Получите в строке
"пять плюс три" 5+3
"двадцать два разделить на семь" 22/7
"открыть скобку пять плюс три закрыть скобку умножить на два" (5+3)*2
"ноль запятая пять" 0.5
Голосовые команды
Числа (поддерживаются до 100)
1-19: один, два, три, ..., девятнадцать
Десятки: двадцать, тридцать, ..., девяносто
Составные: двадцать один, тридцать пять, ...

Архитектура
Проект строго следует паттерну MVVM:
text
Copy
Download
                         VIEW                            
  CalculatorView (PySide6) + ColorPickerDialog          
  - Сетка кнопок, строка формул, диалог выбора цвета     
  - Только отображение и сигналы                         

                      Сигналы

                     VIEWMODEL                           
  CalculatorViewModel                                    
  - Обработка команд от View                             
  - Вызов Model для вычислений                           
  - Управление голосовым вводом                          
  - Загрузка/сохранение цветовых тем                     

                     |         |
                     |         |
              MODEL (данные)    VOICE MODULE   
         CalculatorModel        VoiceInputHandler
         ColorSettings          - Vosk интеграция
         - Безопасное           - CommandParser  
           вычисление           - Русский язык     
         - AST-валидация
Безопасность
Никакого
eval()
— используется безопасный парсинг AST
Валидация узлов — запрещены вызовы функций, атрибуты, имена
Потокобезопасность — голосовой ввод в отдельном потоке с блокировками

[ERROR-1] - Ошибка загрузки модели Vosk
Присутствует в версиях: [v1.0.0] - текущая
Симптомы: При запуске появляется сообщение "Предупреждение: Модель Vosk не найдена", голосовой ввод не работает
Причина: Модель Vosk не скачана или распакована не в ту папку
Решение:
Скачайте модель:
wget https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip
Распакуйте:
unzip vosk-model-small-ru-0.22.zip
Создайте папку
model
в корне проекта
Переместите:
mv vosk-model-small-ru-0.22 model/
[ERROR-2] - Ошибка установки pyaudio на Linux
Присутствует в версиях: [v1.0.0] - текущая
Симптомы:
pip install pyaudio
завершается с ошибкой компиляции
Причина: Отсутствуют системные зависимости portaudio
Решение (Ubuntu/Debian):
bash
Copy
Download
sudo apt-get update
sudo apt-get install portaudio19-dev python3-pyaudio
pip install pyaudio
[ERROR-3] - Микрофон не работает / голос не распознаётся
Присутствует в версиях: [v1.0.0] - текущая
Симптомы: При нажатии кнопки "Голос" ничего не происходит или всегда "Не распознано"
Причина: Микрофон не выбран по умолчанию или низкая громкость
Решение:
Проверьте микрофон:
arecord -l
(должен показывать устройство)
Проверьте уровень звука:
arecord -d 5 test.wav && aplay test.wav
Увеличьте громкость микрофона в настройках системы
В коде можно уменьшить порог чувствительности (строка
if max_amplitude < 500:
в
voice_input_handler.py
)
[ERROR-4] - Голосовая команда не распознаётся правильно
Присутствует в версиях: [v1.0.0] - текущая
Симптомы: Сказали "пять плюс три", а в строке появляется "5 плюс 3" или "пять+три" в неправильном формате
Причина: Vosk распознал слова с ошибками или акцент
Решение:
Говорите чётко, с паузами между словами
Используйте простые команды: "пять плюс три", "двадцать разделить на четыре"
Сложные выражения разбивайте на несколько голосовых вводов
Можно вручную отредактировать строку формул
[ERROR-5] - Вычисление выдаёт "Ошибка в выражении"
Присутствует в версиях: [v1.0.0] - текущая
Симптомы: При нажатии "=" или Enter появляется сообщение об ошибке
Причина: Синтаксическая ошибка (например, два оператора подряд, незакрытые скобки)
Решение:
Проверьте строку формул на правильность
Пример правильного выражения:
(5+3)*2
Пример неправильного:
5++3
или
(5+3
Нажмите "C" для очистки и начните заново
[ERROR-6] - Деление на ноль выдаёт ошибку
Присутствует в версиях: [v1.0.0] - текущая
Симптомы: При делении на ноль появляется сообщение "Деление на ноль"
Причина: Математическая ошибка (деление на ноль)
Решение:
Это нормальное поведение калькулятора
Нажмите "C" для очистки
Введите корректное выражение (делите на ненулевое число)
[ERROR-7] - Калькулятор не запускается на Windows
Присутствует в версиях: [v1.0.0] - текущая
Симптомы: Ошибки при импорте PySide6 или pyaudio
Причина: Проект разработан для Linux, но может работать на Windows с доработками
Решение:
Установите PySide6:
pip install PySide6
Для pyaudio на Windows:
pip install pipwin
→
pipwin install pyaudio
Для Vosk: модель работает на любой платформе
Рекомендуется использовать WSL2 для полной совместимости
[ERROR-8] - Настройки цвета не сохраняются
Присутствует в версиях: [v1.1.0] - текущая
Симптомы: После перезапуска калькулятора цвета кнопок сбрасываются
Причина: Отсутствуют права на запись в файл
color_theme.json
Решение:
Проверьте права на запись в папке проекта:
ls -la color_theme.json
Если файл отсутствует, калькулятор создаст его автоматически при первом изменении цвета
Убедитесь, что у вас есть права на запись:
chmod 755 .
в папке проекта
Если проблема сохраняется, создайте файл вручную:
echo "{\"digit\": \"#FFFFFF\", \"operator\": \"#FFA500\"}" > color_theme.json
[ERROR-9] - Кнопка "Настройки" не открывает диалог выбора цвета
Присутствует в версиях: [v1.1.0] - текущая
Симптомы: Нажатие на кнопку "Настройки" (⚙️) не даёт реакции
Причина: Неправильная настройка сигналов в CalculatorView
Решение:
Перезапустите калькулятор
Убедитесь, что файл
views/color_picker_dialog.py
существует
Проверьте консоль на наличие ошибок импорта
Если ошибка повторяется, переустановите PySide6:
pip install --upgrade PySide6
