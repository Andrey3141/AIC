## 📖 О проекте

**AIC (Artificial Intelligence Company)** — это автоматизированная система, которая имитирует работу IT-компании через 4 роли (Начальник, Аналитик, Главный разработчик, Рядовой разработчик + Тестировщик). Система общается с DeepSeek AI, распределяет задачи, проверяет код, запускает тесты и генерирует документацию без участия человека.

### 🎯 Основная идея

Вместо того чтобы писать код вручную — вы даёте задачу **Начальнику** (через интерфейс AIC), а система автоматически:

1. Уточняет ТЗ через Аналитика
2. Передаёт задание Главному разработчику
3. Генерация кода Рядовым разработчиком
4. Проверка и доработка кода
5. Запуск unit-тестов
6. Обновление документации
7. Отчёт пользователю

### Почему это лучше прямой работы с ChatGPT/DeepSeek?

| Прямой диалог с AI | AIC (ролевая система) |
|--------------------|------------------------|
| Нужно самому формулировать точное ТЗ | Аналитик уточняет требования |
| Код часто с ошибками | Главный разработчик проверяет код |
| Нет тестирования | Автоматический запуск pytest |
| Документацию пишете сами | Аналитик обновляет README, CHANGELOG, FAQ |
| Один ответ — один код | Цикл доработки до 100% качества |

**Полный цикл из 12 этапов:**

1. Пользователь → Начальник (уточнение ТЗ)
2. Начальник → Аналитик (первичное ТЗ)
3. Аналитик → Начальник (уточнённое ТЗ)
4. Начальник → Главный (финальное ТЗ)
5. Главный → Разработчик (задача + файлы)
6. Разработчик → Главный (код)
7. Главный → Тестировщик (запрос тестов)
8. Тестировщик → Главный (тесты + отчёт)
9. Главный → Аналитик (запрос документации)
10. Аналитик → Главный (документация)
11. Главный → Начальник (отчёт)
12. Начальник → Пользователь (докладная)

## 📥 Установка и запуск

### ⚡ Быстрый старт

```bash
# 1. Клонируйте репозиторий (или скопируйте файлы)
cd aic

# 2. Установите зависимости
pip install selenium pyperclip

# 3. Запустите Chromium с отладкой
chromium --remote-debugging-port=9222 --user-data-dir=/tmp/chrome_debug

# 4. Запустите AIC
python main.py
```

### 📦 Требования
- Python 3.8 или выше
- Chromium / Google Chrome (с поддержкой remote debugging)
- Установленный DeepSeek аккаунт (открытый в браузере)

### 🔧 Настройка перед запуском

1. **Создайте папку `prompts/`** с файлами ролей:
   - `Boss.txt`
   - `Analyst.txt`
   - `Chief_developer.txt`
   - `Ordinary_developer.txt`
   - `Tester.txt`

2. **Отредактируйте `aic_config.json`**:
   ```json
   {
       "min_wait_time": 150,
       "max_wait_time": 151,
       "project_folder": "/путь/к/вашему/проекту",
       "test_timeout": 30
   }
   ```

3. **Откройте 5 вкладок DeepSeek** в одном окне Chromium:
   - Вкладка 0: Начальник
   - Вкладка 1: Аналитик
   - Вкладка 2: Главный разработчик
   - Вкладка 3: Рядовой разработчик
   - Вкладка 4: Тестировщик

### 🚀 Запуск

После запуска `python main.py`:
- Нажмите **F11** для полноэкранного режима
- Нажмите **ПОДКЛЮЧИТЬСЯ** для соединения с Chromium
- Укажите папку проекта и настройте тайминги
- Введите задачу в поле чата и нажмите **ОТПРАВИТЬ**

### 👤 Для пользователя (разработчика)

| Шаг | Действие | Что происходит |

## 🔧 Решение проблем

<details>
<summary>❌ Не подключается к Chromium</summary>

1. Убедитесь, что Chromium запущен с флагом:  
   `chromium --remote-debugging-port=9222 --user-data-dir=/tmp/chrome_debug`
2. Проверьте, что порт 9222 свободен: `lsof -i :9222`
3. В настройках AIC порт должен быть 9222
4. Откройте вручную `http://localhost:9222/json` — должен быть ответ
</details>

<details>
<summary>❌ DeepSeek не отвечает</summary>

1. Войдите в DeepSeek в браузере **до запуска AIC**
2. Убедитесь, что открыта именно вкладка чата, а не список диалогов
3. Проверьте, что текстовое поле видимо (textarea)
4. Нажмите вручную "Новый чат" в DeepSeek
</details>

<details>
<summary>❌ Не найден файл промпта</summary>

Создайте папку `prompts/` рядом с `main.py` и положите туда файлы:
- `Boss.txt`
- `Analyst.txt`
- `Chief_developer.txt`
- `Ordinary_developer.txt`
- `Tester.txt`

Содержимое файлов — в документации к проекту.
</details>

<details>
<summary>❌ Тесты не проходят / pytest не найден</summary>

1. Установите pytest в виртуальное окружение: `pip install pytest`
2. Либо создайте venv в `aic/testers/unit_tests/venv`:
   ```bash
   cd aic/testers/unit_tests
   python -m venv venv
   source venv/bin/activate
   pip install pytest
   ```
3. AIC автоматически обнаружит venv и установит недостающие модули
</details>

<details>
<summary>❌ Окна перепутались (не тот AI отвечает)</summary>

- Используйте левую панель → **REFRESH** → выберите нужное окно → **ПЕРЕКЛЮЧИТЬ**
- Убедитесь, что вкладки идут в порядке:  
  0: Начальник, 1: Аналитик, 2: Главный, 3: Рядовой, 4: Тестировщик
</details>
