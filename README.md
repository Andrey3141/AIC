## 📖 Общая концепция

Три проекта объединены общей **идеей автоматизации разработки и классификации без использования сложных нейросетевых решений**, но с разными подходами и целями.

| Проект | Цель | Статус |
|--------|------|--------|
| **v1** | AI-компания через DeepSeek (браузер) | ✅ Завершён |
| **v2** | AI-компания через LM Studio (локальный API) | ✅ Завершён |
| **v3** | Алгоритмический классификатор (без нейросетей) | 🚧 В разработке |

## 🏗 Проект 1: AIC v10.3 (DeepSeek)

### Описание

Первая версия — система, имитирующая работу IT-компании через 5 ролей, общаясь с DeepSeek AI через браузер (Selenium).

### Архитектура

```
ПОЛЬЗОВАТЕЛЬ → НАЧАЛЬНИК → АНАЛИТИК → НАЧАЛЬНИК → 
ГЛАВНЫЙ → РАЗРАБОТЧИК → ГЛАВНЫЙ → ТЕСТИРОВЩИК → 
ГЛАВНЫЙ → АНАЛИТИК → ГЛАВНЫЙ → НАЧАЛЬНИК → ПОЛЬЗОВАТЕЛЬ
```

### Технологии

```yaml
AI:        DeepSeek (через браузер)
Браузер:   Selenium + Chromium (port 9222)
GUI:       Tkinter
Логика:    12-этапный пайплайн
Промпты:   .txt файлы
```

### Установка и запуск

```bash
# 1. Установка зависимостей
pip install selenium pyperclip

# 2. Запуск Chromium с отладкой
chromium --remote-debugging-port=9222 --user-data-dir=/tmp/chrome_debug

# 3. Запуск AIC
python main.py
```

### Требования

- 5 открытых вкладок DeepSeek
- Файлы промптов в папке `prompts/`

## 🏗 Проект 2: AIC v2.0 (LM Studio)

### Описание

Вторая версия — миграция с DeepSeek на локальный LM Studio. Убрана зависимость от браузера, добавлен умный распределитель сообщений.

### Архитектура

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         ПОЛЬЗОВАТЕЛЬ (USER)                             │
└─────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                    РАСПРЕДЕЛИТЕЛЬ (DISTRIBUTOR)                         │
│              Анализирует: ignore / save / send                          │
└─────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                        8 РОЛЕЙ (параллельно)                            │
│  ANALYST → CHIEF_DEVELOPER → ORDINARY_DEVELOPER →                       │
│  CHIEF_TESTER → ORDINARY_TESTER → BOSS → USER                           │
└─────────────────────────────────────────────────────────────────────────┘
```

### Технологии

```yaml
AI:        LM Studio (локальный сервер, порт 1234)
Модель:    Qwen2.5-Coder-7B / qwen3-coder-30b
GUI:       Tkinter
Логика:    Оркестратор + Distributor
Промпты:   .yaml файлы
Формат:    "=== КОМУ === [РОЛЬ]"
```

### Установка и запуск

```bash
# 1. Установка зависимостей
pip install requests pyyaml

# 2. Запуск LM Studio
# - Скачать LM Studio
# - Загрузить модель
# - Запустить сервер (вкладка Developer → Start Server)

# 3. Запуск AIC v2
python main.py
```

### Конфигурация (`config.json`)

```json
{
    "lm_studio_url": "http://localhost:1234/v1/chat/completions",
    "model": "qwen/qwen3-coder-30b",
    "temperature": 0.3,
    "max_tokens": 32768,
    "employees": [...]
}
```

## 🏗 Проект 3: Algorithmic Classifier v3.0

### Описание

Третья версия — **полный отказ от нейросетей**. Чисто алгоритмический классификатор изображений (MNIST) и текста.

### Идеология

| Нейросети | Algorithmic Classifier |
|-----------|------------------------|
| Чёрный ящик | Прозрачные правила |
| Требуют GPU | Работают на CPU |
| Миллионы параметров | Тысячи правил |
| 99.2% на MNIST | **82% на MNIST** |

### Архитектура супер-гибрида

```
ВХОДНОЕ ИЗОБРАЖЕНИЕ (28×28)
           │
           ▼
    CROP TO DIGIT
           │
           ▼
    ПРИЗНАКИ (543D)
    ┌─────────────────────────────────────────────┐
    │ HOG (36D) • LBP (50D) • Моменты (7D)        │
    │ FFT (14D) • DCT (49D) • Nonogram (200D)     │
    └─────────────────────────────────────────────┘
           │
           ▼
    ┌─────────────────────────────────────────────┐
    │ СУПЕР-ГИБРИД (5 модулей)                    │
    │                                              │
    │ 1. Coarse-to-Fine (Оригинальные) [вес:0.31] │
    │ 2. PPN (Координатный)            [вес:0.18] │
    │ 3. PPN (CNN-подобный)            [вес:0.21] │
    │ 4. PPN (Nonogram)                [вес:0.14] │
    │ 5. EditDistance                  [вес:0.16] │
    └─────────────────────────────────────────────┘
           │
           ▼
    ВЗВЕШЕННОЕ ГОЛОСОВАНИЕ
           │
           ▼
    ПРЕДСКАЗАНИЕ (цифра 0-9)
```

### Результаты MNIST

| Модуль | Точность |
|--------|----------|
| Оригинальный (CTF) | 76.2% |
| EditDistance | 74.1% |
| CNN-подобный (PPN) | 71.3% |
| Координатный (PPN) | 68.5% |
| Nonogram (PPN) | 65.8% |
| **СУПЕР-ГИБРИД** | **82.0%** |

### Матрица ошибок (наиболее частые)

| Истинная → Предсказанная | Количество |
|--------------------------|------------|
| 4 → 9 | 147 |
| 7 → 1 | 89 |
| 3 → 8 | 76 |
| 5 → 6 | 68 |

> Это те же ошибки, что делает человек!

### Установка и запуск

```bash
# 1. Установка зависимостей
pip install numpy scipy scikit-image matplotlib

# 2. Запуск меню
python main.py

# 3. Выбор теста
#   1 - Логическая задача
#   2 - Логическая задача (5 чисел)
#   3 - MNIST (Супер-гибрид)
#   4 - Генерация текста (в разработке)
```

## 📈 Результаты и достижения

### Сравнение точности

| Задача | v1 | v2 | v3 | Нейросеть (бенчмарк) |
|--------|----|----|-----|----------------------|
| MNIST | ❌ | ❌ | **82%** | 99.2% |
| Логические числа | ✅ | ✅ | ✅ (~100%) | ~100% |
| Генерация текста | ✅ | ✅ | 🚧 | Высокое качество |

### Ключевые выводы

1. **82% на MNIST алгоритмами** — это прорыв. Без единой нейронной сети, только признаки + парные паттерны.
2. **Рекурсия не работает** — ошибки уровня 1 становятся сложнее, переобучение на малом количестве данных.
3. **LM Studio быстрее DeepSeek** — убрана задержка на таймеры и Selenium.
4. **Прозрачность имеет цену** — мы знаем, почему модель ошибается (4↔9, 7↔1).

## 🔧 Решение проблем

### Общие для всех версий

> [ERR-COMMON-001] - Ошибка импорта модуля

### Присутствует в версиях: [v1, v2, v3]
### Симптомы: ModuleNotFoundError
### Решение:
```bash
# v1
pip install selenium pyperclip

# v2
pip install requests pyyaml

# v3
pip install numpy scipy scikit-image matplotlib
```

---

### Версия 1 (DeepSeek)

> [ERR-V1-001] - Не подключается к Chromium

### Решение:
```bash
chromium --remote-debugging-port=9222 --user-data-dir=/tmp/chrome_debug
# Проверка: http://localhost:9222/json
```

> [ERR-V1-002] - Не найден файл промпта

### Решение: Создать папку `prompts/` с файлами:
- Boss.txt, Analyst.txt, Chief_developer.txt, Ordinary_developer.txt, Tester.txt

---

### Версия 2 (LM Studio)

> [ERR-V2-001] - Не подключается к LM Studio

### Решение:
1. Запустить LM Studio
2. Загрузить модель
3. Вкладка Developer → Start Server
4. Проверить порт 1234

> [ERR-V2-002] - Распределитель не отправляет сообщение

### Решение: Проверить формат — должно быть "=== КОМУ === [РОЛЬ]"

> [ERR-V2-003] - Модель не найдена

### Решение: Указать точное имя модели в `config.json`

---

### Версия 3 (Algorithmic Classifier)

> [ERR-V3-001] - Низкая точность MNIST (<70%)

### Решение: Увеличить max_pairs в super_hybrid.py
```python
model.fit(X_train, y_train, max_pairs=500000, min_support=5)
```

> [ERR-V3-002] - Долгое обучение (>10 минут)

### Решение: Уменьшить max_pairs или количество признаков

> [ERR-V3-003] - Тест 4 не работает

### Решение: В разработке (v3.1.0). Используйте тесты 1-3.
